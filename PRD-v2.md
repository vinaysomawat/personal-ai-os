# Product Principles

## Principle 1

Automation over manual work.

If something can be automated, automate it.

---

## Principle 2

Rule Engine before AI.

Before using AI always ask

Can deterministic code solve this?

If yes,

do not call AI.

---

## Principle 3

AI is expensive.

Anthropic credits cost money.

Treat AI like a premium feature.

Never call AI for

* calculations
* sorting
* filtering
* score calculation
* dashboards
* reminders
* charts
* notifications
* simple recommendations

Use AI only for

* mentoring
* coaching
* reviewing
* explaining
* brainstorming
* summarizing
* generating plans

Every unnecessary AI request is considered a bug.

---

## Principle 4

Every AI request must pass through one AI Gateway.

The gateway should

* choose the cheapest suitable model
* check cache
* check AI budget
* check rule engine
* check usage limits
* log cost
* provide graceful fallback

No module should directly call Anthropic.

---

## Principle 5

Everything should contribute toward goals.

Avoid isolated modules.

Learning improves Career.

Health improves Productivity.

Productivity improves Coding.

Coding improves Career.

Everything contributes toward overall growth.

---

## Principle 6

The application should reduce decisions.

Instead of showing me 25 tasks,

tell me the three highest-impact actions.

---

## Principle 7

Every page must answer

* What happened?
* Why?
* What should I do next?

# When Suggesting New Features

Do not add features because they are interesting.

Every feature must satisfy all of these conditions:

* Solves a real problem.
* Saves time.
* Reduces manual work.
* Integrates with existing modules.
* Improves long-term consistency.
* Works with minimal AI usage.
* Is maintainable.
* Is reusable.
* Can be measured.

If it does not satisfy these requirements,

do not build it.

# Before Building Any Feature

Always perform this checklist.

1. Is AI actually required?

2. Can deterministic code solve it?

3. Can existing modules be reused?

4. Does this increase productivity?

5. Does this reduce manual effort?

6. Does this improve one of my goals?

7. Is this worth maintaining for years?

8. Is there a simpler solution?

Only after answering these questions should implementation begin.
